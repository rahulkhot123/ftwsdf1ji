SCP Commands in Linux

scp Command stands for Secure Copy Command. To copy files between two systems in the secure way the scp command used. For security, this command encrypts files and passwords sent to another remote or local system. The scp command mainly used to transfer files from one remote system to another local system or from one remote system to another remote system. In this tutorial, you are going to learn scp command in Linux to securely transfer files.

Basic Syntax

The basic syntax for scp command given below:

scp [OPTION] [username@]source_host:filename_1 [username@]dest_host:filename_2
In the above syntax:

filename_1: Source filename on the remote or local server

filename_2: Destination filename or directory path on the remote or local server

[OPTION]: Options for scp command are looks like below:

  • -r: This option will copy directory recursively.
  • -p: It will preserve the file access and modification time data.
  • -C: Force scp to compress data before sending it.
  • -P: Use custom remote host ssh port.
  • -v: Verbose mode.
  • -c: Specify cipher for data encryption.

SCP Command Examples

Following are the SCP command examples:

1. Copy file from remote system to local system using SCP command.

Let's assume you want to copy or transfer the file demo.txt from a remote host system to local host system then you should run following command in the terminal: 
scp remote_username@202.20.x.x:/tmp/demo.txt ~/Documents

In the above example, the file named demo.txt will be copied from remote system to local system inside Documents directory.

2. Copy file from a local system to remote system using SCP command.

Now if you want to copy or transfer the file demo.txt from a local system to remote system then type following command in the terminal and hit Enter key: 
scp /Desktop/demo.txt remote_username@202.20.x.x:/tmp

In the above example, the file named demo.txt will be copied from local system to remote system inside /tmp directory.

3. Copy file from remote system to another remote system using SCP command.

Let's assume if you want to copy or transfer the file demo2.txt from a remote host system to another remote host system then you should run following command in the terminal: 
scp remote_username_1@202.20.x.x:/tmp/demo.txt remote_username_2@example.edu:/opt

In the above example, the file named demo.txt inside /tmp directory will be copied from one remote system to another remote system inside /opt directory.

4. Copy directory from remote system to local system using SCP command.

Here we want to copy the whole directory named test1 from a remote host system to local host system then you should run following command in the terminal: 
scp -r remote_username@202.20.x.x:/tmp/test1 ~/Documents

In the above example, the directory named test1 inside /tmp directory will be copied from one remote system to local system inside Documents directory as like below:

5. Copy files from a local system to remote system using SCP command.

Now here we want to transfer the file demo1.txt and demo2.txt from a local system to remote system then we should type following command in the terminal and hit Enter key: 
scp demo1.txt demo2.txt remote_username@202.20.x.x:/tmp

In the above example, the file named demo1.txt and demo2.txt will be copied from local system to remote system inside /tmp directory .

6. Copy files from remote system to local system using SCP command.

Let's assume you want to copy or transfer the file demo1.txt and demo2.txt from a remote host system to local host system then you should run following command in the terminal: 
scp remote_username@202.20.x.x:/tmp/\{demo1.txt,demo2.txt\} ~/Documents

In the above example, the file named demo1.txt and demo2.txt will be copied from remote system to local system inside Documents directory.

Performance Improvement

We can also improve the performance of scp command in Linux by using different cipher for encryption. In scp command Triple-DES cipher is used for encryption of data to be sent.

Here to improve the performance we can use blowfish cipher to encrypt the data.

scp -c blowfish /Desktop/demo.txt remote_username@202.20.x.x:/tmp
